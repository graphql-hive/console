# Getting Started

This guide provides a walkthrough of setting up the Hive Router. We'll cover everything from the initial installation to running a live, queryable GraphQL API on your local machine. By the end, you'll have a running instance of the router that serves a federated supergraph composed of several subgraphs.


## Installation

Hive Router is distributed as a self-contained binary and a Docker image, giving you the flexibility to run it in any environment. For local development or quick tests, the binary is often the fastest way to get started. For production or containerized environments, the Docker image is the recommended approach.


<Tabs items={["Binary", "Docker"]} >

{/* Binary */}

<Tabs.Tab>

This command will download the appropriate binary for your operating system.

```sh
curl -o- https://raw.githubusercontent.com/graphql-hive/router/main/install.sh | sh
```

> At the moment, pre-compiled binaries are only available for Linux and MacOS. If you are using Windows, please use the Docker option.

</Tabs.Tab>

{/* Docker */}

<Tabs.Tab>

You can use the official Docker image to run Hive Router.

```sh
docker pull ghcr.io/graphql-hive/router
```

</Tabs.Tab>

</Tabs>

For a full list of releases and specific versions, see the [GitHub Releases](https://github.com/graphql-hive/router/releases) page.

## Supergraph Schema

To follow this guide, you will need to download a sample supergraph schema. This schema is the same one used in the ["Get Started with GraphQL Federation"](docs/schema-registry/get-started/apollo-federation#prerequisites) guide.

It is composed of three publicly available subgraphs: [users](https://federation-demo.theguild.workers.dev/users), [products](https://federation-demo.theguild.workers.dev/products) and [reviews](https://federation-demo.theguild.workers.dev/reviews).

To download the supergraph schema, run the following command in your terminal:

```sh
curl -sSL https://federation-demo.theguild.workers.dev/supergraph.graphql > supergraph.graphql
```

Once downloaded, you will provide this schema file to the router.

> **What is a supergraph?** A supergraph is a special schema that combines all of your individual subgraph schemas into a single, federated graph. The router uses this supergraph to understand how to route incoming requests to the appropriate subgraphs.

## Basic Configuration

The router needs to know where to find your supergraph schema file. Create a simple configuration file named `router.config.yaml` in your project directory.

```yaml filename="router.config.yaml"
supergraph:
  source: file
  path: ./supergraph.graphql
```

Now you have a basic configuration that tells the router to load the supergraph schema from the `supergraph.graphql` file that you downloaded earlier.

## Running the Router

With your configuration file in place, you can now start the router.

<Tabs items={["Binary", "Docker"]} >

{/* Binary */}

<Tabs.Tab>

This command will start the router using the configuration file you created earlier.

```sh
./hive_router
```

</Tabs.Tab>

{/* Docker */}

<Tabs.Tab>

You can pass configuration to the container using environment variables. You will also need to mount your supergraph file into the container.

```bash
docker run \
  -p 4000:4000 \
  -v ./supergraph.graphql:/app/supergraph.graphql \
  -v ./router.config.yaml:/app/router.config.yaml \
  ghcr.io/graphql-hive/router:latest
```

</Tabs.Tab>

</Tabs>

If the router starts successfully, you will see log output indicating that the server is listening on port `4000`.

You can now explore the API using the built-in GraphQL IDE. Open `http://localhost:4000` in your browser and try running the following query to see the router in action.

```graphql
query TopProducts {
  topProducts {
    name
    reviews {
      body
      author {
        name
      }
    }
  }
}
```
