# Introduction to Hive Router

Hive Router is a high-performance, open-source [GraphQL Federation](/federation) router written in Rust. It acts as the central entry point for all incoming GraphQL requests, efficiently planning and executing them across your distributed subgraph services.

## Why Hive Router?

In a [federated architecture](/federation), the router is a critical piece of infrastructure. Its performance and reliability directly impact your entire API. Hive Router is designed to meet the demands of modern, large-scale GraphQL applications.

- **Blazing Fast:** Built with Rust, Hive Router is designed for maximum performance and low resource consumption, ensuring minimal latency overhead for your API calls.
- **Fully Open-Source:** Licensed under MIT, you can use, modify, and deploy it anywhere without restrictions.
- **Robust and Reliable:** Leverage the safety and concurrency guarantees of Rust for a stable and resilient gateway.
- **Modern Federation Features:** Supports the latest GraphQL Federation standards and provides powerful tools for configuration, security, and traffic management.

To learn more about the motivation and vision behind this project, [read our introductory blog post](https://the-guild.dev/graphql/hive/blog/welcome-hive-router).
