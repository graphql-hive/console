# Security

Hive Router is designed with security in mind. It provides several features to help you secure your GraphQL API and protect it from common vulnerabilities. This section provides an overview of the security features available in Hive Router.

## Authentication (JWT)

Hive Router supports JWT-based authentication to control access to your GraphQL API. You can configure the router to validate JWTs from incoming requests and reject unauthorized access.

[Learn more about Authentication (JWT)](./authentication)

## Cross-Origin Resource Sharing (CORS)

CORS is a security feature that allows you to control which origins are allowed to access your API. Hive Router provides a flexible CORS configuration that allows you to define different policies for different origins.

[Learn more about CORS](./cors)

## Cross-Site Request Forgery (CSRF)

CSRF is an attack that tricks a user into performing an unwanted action on a web application in which they're currently authenticated. Hive Router provides CSRF protection to prevent such attacks.

[Learn more about CSRF](./csrf)
